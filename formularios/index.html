🛡️ Validación de Formularios en JavaScript

¿Alguna vez intentaste registrarte en una página web y, al dejar un espacio vacío o poner un correo falso, el sistema te mostró un mensaje como "Por favor ingresa un correo válido"? Eso es gracias a la validación de formularios ✨. Sin esta herramienta, cualquiera podría enviar información incompleta o incorrecta: un correo inventado, una contraseña de un solo carácter o incluso dejar todo en blanco.

La validación es como un filtro inteligente 🔍 que ayuda a los programadores a asegurarse de que los usuarios escriban lo que realmente se necesita. Y lo mejor es que JavaScript nos permite hacerlo de manera dinámica, rápida y visible, sin necesidad de recargar la página.

En esta unidad vas a aprender cómo hacer que tus formularios sean inteligentes, ¡como si tuvieran un guardián que revisa la entrada de cada usuario! 🚀

Objetivos de aprendizaje

  • 🧠 Comprender la importancia de la validación de formularios para la seguridad y la experiencia del usuario en aplicaciones web.
  • 🔧 Aplicar JavaScript para implementar validaciones dinámicas en formularios web, verificando datos en tiempo real.
  • ✅ Desarrollar formularios interactivos y funcionales que promuevan buenas prácticas de programación y usabilidad.

A continuación encontrarás un tutorial interactivo completo sobre validación de formularios en JavaScript. Explora cada sección para aprender cómo verificar datos y crear formularios seguros.

💡 Consejo: Tómate tu tiempo para explorar cada sección del tutorial. Practica los ejemplos y experimenta con validaciones para reforzar tu aprendizaje.

👥 Actividad 1: Formulario de registro para red social

Imagina que estás diseñando el formulario de registro para una nueva red social dirigida a jóvenes. El formulario debe permitir que una persona cree su cuenta ingresando su nombre completo, su correo electrónico y una contraseña. Sin embargo, para que el registro sea válido se deben cumplir las siguientes condiciones:

  • • El nombre no puede estar vacío (no se permiten registros anónimos).
  • • El correo electrónico debe tener un formato correcto (por ejemplo: usuario@dominio.com).
  • • La contraseña debe tener mínimo 8 caracteres, ya que es un requisito de seguridad de la plataforma.

Analiza el problema, identifica los datos de entrada, las restricciones que se deben aplicar a cada campo y cuál sería la salida esperada (ejemplo: mensaje de registro exitoso o advertencia de error).

💡 Pista: Valida que el nombre no esté vacío, usa expresiones regulares para el correo y verifica la longitud de la contraseña con .length.

🎓 Actividad 2: Inscripción a cursos virtuales

El colegio virtual "Aprende en Casa" quiere automatizar el proceso de inscripción a sus cursos. Para eso, el estudiante debe llenar un formulario en línea con su edad y el curso que desea inscribir. Para que la inscripción sea válida, debes cumplir con estas reglas:

  • • La edad no puede estar vacía y debe ser mayor o igual a 14 años (ya que los cursos están diseñados para adolescentes y jóvenes).
  • • El estudiante debe seleccionar un curso real de la lista desplegable. La opción por defecto "Selecciona una opción" no puede ser considerada como válida.

A partir de este enunciado, identifica los campos de entrada, qué restricciones tiene cada uno y cuál sería la salida (ejemplo: confirmación de inscripción o mensaje de error indicando qué falta).

💡 Pista: Verifica que la edad sea un número mayor o igual a 14 y que el valor del select no sea la opción por defecto.

💳 Actividad 3: Sistema de pago con tarjeta

Una tienda virtual de videojuegos implementa un sistema de pago con tarjeta de crédito o débito. El usuario debe ingresar los siguientes datos en el formulario: número de tarjeta, fecha de vencimiento y código de seguridad (CVV). Para garantizar que los pagos sean procesados correctamente, el sistema valida lo siguiente:

  • • El número de tarjeta debe contener exactamente 16 dígitos numéricos.
  • • La fecha de vencimiento no puede estar vacía, ya que es un dato esencial para validar la tarjeta.
  • • El código de seguridad (CVV) debe tener exactamente 3 dígitos.

Identifica los campos de entrada, sus restricciones y la salida que debe mostrar el sistema (ejemplo: pago procesado con éxito o mensaje indicando el error específico).

💡 Pista: Usa .length para verificar la cantidad exacta de dígitos y valida que sean solo números.
